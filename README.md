# ZeroMQ Сокеты для Android

## Описание
Приложение демонстрирует работу с сетевыми сокетами через библиотеку ZeroMQ — реализован локальный REP-сервер и два типа клиентов: внутренний (localhost) и внешний (по IP-адресу).

## Функциональность

* **Локальный сервер**: Запуск REP-сервера на порту 5556, приём сообщений и отправка ответов.
* **Внутренний клиент**: Подключение к серверу через `localhost` и обмен сообщениями внутри устройства.
* **Внешний клиент**: Подключение к удалённому серверу по IP-адресу и порту.
* **Управление состоянием**: Потокобезопасное переключение запуск/остановка сервера и клиента через `AtomicBoolean`.
* **Лог сообщений**: Отображение всех событий с временными метками в реальном времени.

## Используемые компоненты UI

* `TextView` — для отображения статуса соединения и лога сообщений.
* `Button` — для запуска/остановки сервера, внутреннего и внешнего клиентов.

# GPS-локация с отправкой на сервер

## Описание
Приложение получает GPS-координаты устройства в реальном времени, сохраняет их локально в JSON-файл и отправляет на удалённый сервер через ZeroMQ.

## Функциональность

* **Получение координат**: Непрерывное отслеживание местоположения через `FusedLocationProviderClient` с высокой точностью.
* **Отправка на сервер**: Передача координат (широта, долгота, высота, время) на C++ сервер по протоколу ZMQ REQ/REP.
* **Локальное сохранение**: Запись каждой точки маршрута в файл `location_history.json` во внутренней памяти.
* **Запрос разрешений**: Автоматический запрос `ACCESS_FINE_LOCATION` и `ACCESS_COARSE_LOCATION` при запуске.
* **Обработка ошибок**: Таймауты соединения, переподключение при разрыве связи.

## Используемые компоненты UI

* `TextView` — для отображения широты, долготы, высоты, времени и статуса ZMQ-соединения.

# Фоновый сервис сбора сотовых вышек

## Описание
Фоновый Foreground Service, который одновременно собирает GPS-координаты и данные о всех доступных сотовых вышках (GSM, LTE, 5G NR), отправляет их на сервер и рассылает обновления через `LocalBroadcast`.

## Структура проекта
- **CellLocationService.kt**: Foreground Service — ядро приложения. Периодически (каждые 5 секунд) собирает данные о местоположении и сотовых вышках через `TelephonyManager`, формирует JSON, отправляет его на ZMQ-сервер, записывает в файл и рассылает broadcast-обновление.
- **CellLocationServiceActivity.kt**: Activity для управления сервисом. Содержит кнопки запуска/остановки и отображает получаемые данные о вышках через `BroadcastReceiver`.

## Функциональность

* **Сбор данных о вышках**: Получение информации по стандартам GSM (LAC, CID, ARFCN, уровень сигнала), LTE (PCI, TAC, EARFCN, RSRP, RSRQ) и 5G NR (NCI, NR-ARFCN, SS-RSRP, SS-RSRQ, SS-SINR).
* **Фоновая работа**: Сервис продолжает работу при свёрнутом приложении, отображая постоянное уведомление.
* **Отправка на сервер**: Полный JSON с координатами, данными вышек и статистикой трафика отправляется через ZMQ.
* **Локальное сохранение**: Данные дописываются в файл `location_history.json`.
* **Broadcast-обновления**: Активность получает данные в реальном времени через `LocalBroadcastManager`.

## Используемые компоненты UI

* `TextView` — для отображения статуса сервиса и данных о сотовых вышках.
* `Button` — для запуска и остановки фонового сервиса.
